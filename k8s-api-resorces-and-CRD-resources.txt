When dealing with Rook and talk about extending K8s API resources then learned CRD (Custom Resource Defination)

To see it execute command below ; 

kubectl get crd --all-namespaces

Not : --all-namespaces is optional and depend

You wonder, okay what is the already existing resource related K8s version ? Then related command come to rescue. 

kubectl api-resources
